# AegisIDE v3.2.0 Release Notes

## 🚀 Router-First NLU/NLP Autonomy

Transform your AI assistant into a truly autonomous teammate that understands natural language and works without permission prompts.

### 🧠 What's New

**NLU/NLP Intent Recognition**
- Natural language commands: "yes", "go ahead", "run next", "full test"
- Multilingual support: Spanish, French, German, Hindi variants
- Automatic workflow mapping with confidence scoring
- Runtime logging in `activeContext.json.nlu`

**Zero Permission Prompts (0–99%)**
- Router enforces autonomous execution without asking
- Banned phrases auto-corrected with RL penalties
- High-risk actions still gated by risk classifier
- Communication rules prevent stalling behaviors

**Autonomous Workflow Activation**
- Auto-triggers: queue present → `/next`, empty → `/status`, bootstrap needed → `/bootstrap`
- Workflow chaining: `/full_test` → `/status` → `/validate` → `/update` → `/memory-status`
- Retry/backoff with failure logging to `mistakes.json`

**Risk Classification**
- Keyword-based high/low risk detection
- Smart routing: low-risk auto-executes, high-risk requires approval
- Configurable thresholds and decision policies

### 🔧 Technical Improvements

**Workflow Hardening**
- All 12 workflows use router-first `@mcp:json-jq` reads
- Atomic writes via `jq | sponge` (80x faster)
- Removed CLI dependencies: `bc` → Python calculations
- Fixed validation logic to use data files, not schema names

**Memory Bank Policy**
- JSON files are autogenerated and ignored in git
- Schemas in `src/.aegiside/schemas/` are the tracked source of truth
- Clean repository with no live state committed
- Initialize locally via `/init` workflow
